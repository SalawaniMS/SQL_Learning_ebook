Basic Knowledge and Introduction to SQL

  • SQL functions fit into two broad categories:
    • Data Definition Language (DDL): create database objects like tables, indexes, views; define access rights.
    • Data Manipulation Language (DML): insert, update, delete, retrieve data.
  • Database: shared structure containing end-user data and metadata stored in tables.
  • Before using new DBMS:
    • Create database structure to hold tables
    • Create tables for end-user data
  • Two tasks:
    • Create database structure
    • Create tables to hold data
  • RDBMS creates physical files; differs between systems
  • Selection dictated by data nature and intended use.
  • Important for sorting and data retrieval.
  • One line per column definition
  • Line up attributes and constraints
  • Names capitalized; NOT NULL, UNIQUE, primary/foreign keys
  • Command ends with semicolon
CREATE DATABASE <database_name>;
DROP TABLE <table_name>;
CREATE TABLE <table_name> (
    attribute1_name data_type1 [NOT NULL] [UNIQUE],
    attribute2_name data_type2 [NOT NULL] [UNIQUE],
    PRIMARY KEY(attribute_name),
    FOREIGN KEY(attribute_name) REFERENCES <table_name>
);
  • Integer → INTEGER, Small Integer → SMALLINT
  • Numeric → NUMERIC(l,d), Decimal → DECIMAL(l,d)
  • Character → CHAR(l), VARCHAR(l)
  • Date → DATE
  • Primary key creates unique index automatically
  • CREATE INDEX for selected attribute
  • Composite index: two or more attributes, prevent duplication
  • Adding, updating, deleting table rows
  • Saving/restoring table contents
  • Insert rows with select subquery
INSERT INTO <table_name> 
VALUES (attribute1_value, attribute2_value, ...);
